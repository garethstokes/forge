# forge-0m6 — Generated-on-update lifecycle (`Touched` marker)

**Status:** Approved design (brainstorm complete) · **Date:** 2026-06-28
**Bead:** forge-0m6 (follow-ups to forge-2am)

---

## Problem

forge-2am landed the field-policy machinery (markers → distinct Create/Update/Read
projections, `Generated`, `Patch`, `assignments`, snapshot-diff `flushSave`). Four
items were deferred. This bead resolves them:

1. **Generated-on-update stamping.** `touchGenerated` was a no-op. `updated_at`-style
   columns — generated by the DB, but *re-stamped on every UPDATE* — have no
   representation. `Generated` is insert-only (`DEFAULT now()` fills it once; never
   touched again), so it can't express "bump on every write".
2. **RETURNING refresh** (insert side).
3. **Secret/Masked codec wiring** into entity JSON derivation.
4. **`Generated` insert-vs-update distinction** (created_at once vs updated_at every write)
   — this is the same gap as (1), framed differently.

## Scope decision

- **Build:** items (1) + (4) — they are one feature: a marker for
  generated-on-*every*-update columns. Call it **`Touched`**.
- **Item (2) insert side: already DONE.** `insertCreate` issues
  `renderInsert … RETURNING <all cols>`, decodes the row, and `setBaseline`s the
  refreshed entity. No work needed; mark done.
- **Item (2) update side:** folded into this feature as **stamp-only** (below) — no
  RETURNING/refresh on UPDATE.
- **Item (3) Secret/Masked JSON wiring: DESCOPED.** manifest has no entity-level
  JSON derivation layer to hook into — only `Json`/`Aeson` *column* wrappers and a
  standalone `Masked` primitive. Wiring `Secret` into auto-masked entity JSON needs a
  derivation layer that does not exist; out of scope. (The `Masked` primitive remains
  available for manual use.) File a follow-up if entity-JSON derivation is ever added.

## Decision

A new marker **`Touched a`**: generated by the DB on insert AND re-stamped on every
UPDATE. `Generated a` stays insert-only. The two differ in exactly one behaviour —
whether an UPDATE re-stamps the column — and are otherwise identical from the app's
view (the app never reads-to-write or writes them).

Stamping uses the DB transaction clock (`now()`), appended by `renderUpdate`.
**Stamp-only:** no `RETURNING`, no in-memory refresh. Because `Touched` columns are
omitted from the Create/Update projections and skipped in the `flushSave` diff (they
carry `fieldIsGenerated = True`), an in-memory value going stale after an UPDATE is
harmless — nothing reads it back into a write path.

## Design

### Marker + projections (`Manifest/Core/Table.hs`)

- `data Touched a` (phantom marker, like `Generated`).
- `Base (Touched a) = Base a` — stripped on the Read/Identity projection.
- `Field Create (Touched a) = Omitted` — DB `DEFAULT now()` fills it on insert.
- `Field Update (Touched a) = Omitted` — the app never sets it; the DB re-stamps.
- Read projection: `Base a` (via the `Identity` clause), so it's visible when reading.

Projections are **identical to `Generated`**. The only divergence is the UPDATE
auto-stamp, carried by metadata (below), not by the type family.

### Metadata

- **`FieldMeta` gains `fieldTouchedOnUpdate :: Bool`** with a **class-level default
  `= False`** — only the `Touched` instance overrides it, so existing `FieldMeta`
  instances need no change.
- The `Touched` `FieldMeta` instance sets `fieldIsGenerated = True` (⇒ omitted from
  Create *and* skipped in the `flushSave` diff, exactly like `Generated`) **and**
  `fieldTouchedOnUpdate = True`.
- **`ColumnMeta` gains `cmTouchedOnUpdate :: Bool`**, populated by `GColumns` from
  `fieldTouchedOnUpdate @t`.

### Rendering (`Manifest/Core/Sql.hs`)

`renderUpdate` auto-appends the touched columns as literal `= now()` assignments,
read from the table meta:

```haskell
renderUpdate tm setCols pkCol =
  let sets    = [ c <> " = " <> placeholder i | (c, i) <- zip setCols [1 ..] ]
      touched = [ cmName c <> " = now()" | c <- tmColumns tm, cmTouchedOnUpdate c ]
      pkPh    = placeholder (length setCols + 1)
  in "UPDATE " <> tmTable tm
       <> " SET " <> bcIntercalate ", " (sets ++ touched)
       <> " WHERE " <> pkCol <> " = " <> pkPh
```

- `now()` is a literal expression → consumes **no** parameter → the PK placeholder
  stays at `length setCols + 1`. No call-site changes.
- Both real callers benefit for free: `flushSave` (`Session.hs`) and
  `Command.update` (→ `patch`).
- Touched columns never appear in `setCols` (Omitted in the Update projection, skipped
  in the diff), so there is no duplicate-assignment risk.

### Why auto-append in `renderUpdate` (not at call sites)

`renderUpdate` already receives the `TableMeta`, so it has everything it needs; doing
it there means every UPDATE path stamps consistently with zero call-site churn. The
cost is that `renderUpdate` now reads `tm` for touched columns — documented inline.

## Edge cases

- **Empty patch / empty diff with a touched column.** `Command.update key []` would
  render `SET updated_at = now() WHERE pk = $1` — a no-op change that still bumps the
  timestamp. Acceptable (a no-field patch is degenerate). `flushSave` avoids it via its
  existing empty-diff short-circuit (it only issues an UPDATE when the diff is
  non-empty).
- **Entity with no touched column** (e.g. the existing `User` fixture). `touched = []`,
  so `renderUpdate` output is byte-identical to today — the existing `SqlSpec`
  assertion stays green.

## Testing / verification

A fixture entity with a `Touched UTCTime` column (e.g. `updated_at`) alongside a
`Generated UTCTime` `created_at`:

1. **Projection proofs** (type-level): `Field Create (Touched UTCTime) ~ Omitted` and
   `Field Update (Touched UTCTime) ~ Omitted`.
2. **Render:** `renderUpdate` for that table includes `updated_at = now()` in the SET
   clause, and the PK placeholder index is unchanged.
3. **Integration** (`withEmptyDb`): insert a row; UPDATE a normal column; assert
   `updated_at` advanced AND `created_at` (a `Generated` col) did **not** change.

## Out of scope / follow-up

- Secret/Masked auto-masking in entity JSON — needs an entity-JSON derivation layer
  that manifest does not have. File when/if that layer is built.
