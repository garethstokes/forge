**Goal:** Make HealthBench's balanced-F1 (mean of met-class and not-met-class F1) a native output of the calibration harness — computed in crucible, persisted on `MetaEval`, shown on the dashboard.

**Architecture:** Add `passF1`/`failF1`/`balancedF1` to crucible's `CalibrationReport` (pure, from the existing verdict pairs). - [ ] **Step 2: Add the `harmonic` helper** (top-level, near `kappaOf`):
```haskell
-- | F1 = harmonic mean of precision and recall; 0 when both are 0.
harmonic :: Double -> Double -> Double
harmonic p r = if p + r == 0 then 0 else 2 * p * r / (p + r)
```

- [ ] **Step 3: Compute the F1s in both report builders**

In `reportFrom` (the 5-tuple `judged` form), after the `fRec` line add:
```haskell
    jPass  = [h' | (_, h', True, _, _) <- judged]   -- judge said met
    pPrec  = ratio (length (filter id jPass)) (length jPass) 1
    hPass  = [p' | (_, True, p', _, _) <- judged]    -- human said met
    pRec   = ratio (length (filter id hPass)) (length hPass) 1
    passF1_     = harmonic pPrec pRec
    failF1_     = harmonic fPrec fRec
    balancedF1_ = (passF1_ + failF1_) / 2
```
and append the three to the positional constructor:
```haskell
  CalibrationReport po kap fPrec fRec cont errs exampleCount_ measured_ ci abst passF1_ failF1_ balancedF1_
```

In `reportFromVerdicts` (the `(h, j)` form), after `fRec` add:
```haskell
    jPass  = [h | (h, True) <- judged]   -- judge said met
    pPrec  = ratio (length (filter id jPass)) (length jPass) 1
    hPass  = [j | (True, j) <- judged]   -- human said met
    pRec   = ratio (length (filter id hPass)) (length hPass) 1
    passF1_     = harmonic pPrec pRec
    failF1_     = harmonic fPrec fRec
    balancedF1_ = (passF1_ + failF1_) / 2
```
and append to its constructor:
```haskell
  CalibrationReport po kap fPrec fRec [] errs 0 (length judged) ci [] passF1_ failF1_ balancedF1_
```

- [ ] **Step 5: Add a test in crucible `test/Spec.hs`**

Find the existing calibration test block. Add an assertion using `reportFromVerdicts` over a hand-built verdict list with a known confusion matrix. Example: 2 both-met, 1 both-not-met, 1 judge-met-human-not (FP for met class). With cases `[("a",True,Just True),("b",True,Just True),("c",False,Just False),("d",False,Just True)]`:
- met class: predicted-met = {a,b,d}=3, actual-met={a,b}=2, TP_met={a,b}=2 → pPrec 2/3=0.667, pRec 2/2=1.0, passF1 = 2·0.667·1/(1.667)=0.8
- not-met class: predicted-not-met={c}=1, actual-not-met={c,d}=2, TP={c}=1 → fPrec 1/1=1.0, fRec 1/2=0.5, failF1 = 2·1·0.5/1.5=0.667
- balancedF1 = (0.8+0.667)/2 = 0.733

Add (matching the file's assertion style — likely `expect`/`shouldBe`; mirror neighbours):
```haskell

            [ ("a", True,  Just True), ("b", True, Just True)
            , ("c", False, Just False), ("d", False, Just True) ]
  -- met-class F1 0.8, not-met-class F1 ~0.667, balanced ~0.733
  expectClose "passF1"     r.passF1     0.8
  expectClose "failF1"     r.failF1     0.6666666666666666
  expectClose "balancedF1" r.balancedF1 0.7333333333333334
```
(Use whatever near-equality / exact-equality helper the crucible test file uses; if it only has exact `==`, assert the exact `Double` values shown. Read the file's conventions first.)
